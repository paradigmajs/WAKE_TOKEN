# Security Policy

The WAKE smart contract system is designed with security as a primary priority.

The contracts have been built using industry standard libraries and patterns.

## Security Design

Key protections include:

- fixed token supply
- vault-based distribution
- strict access control
- merkle proof verification
- controlled token emissions
- vesting enforcement

All contracts rely on OpenZeppelin implementations where applicable.

## Reporting Vulnerabilities

If you discover a security vulnerability, please report it privately.

Contact:
security@wake-project.io

Do not disclose vulnerabilities publicly before they are addressed.

## Audit

The project is undergoing a third-party security audit prior to mainnet deployment.
